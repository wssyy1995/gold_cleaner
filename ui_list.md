## ⚠️ 技术规格说明

### 尺寸规范

- **逻辑像素**：文档所有尺寸均为游戏内逻辑像素（640x960基准）
- **物理像素**：云存储上传时需提供 **2倍DPR版本**（即1280×1920）
- **适配建议**：使用scale缩放适配不同屏幕，不要写死尺寸值

### 文件格式

| 格式 | 优点 | 使用场景 |
|------|------|--------|
| **PNG** | 支持透明，质量高，兼容好 | UI元素、工具、污垢、特效 |
| **WebP** | 压缩率高，文件小 | 背景图、大纹理（减少加载时间） |
| **JPG** | 文件小，加载快 | 背景纹理、装饰元素（无透明需求） |

### 优先级说明

| 优先级 | 定义 | 处理时间 |
|--------|------|--------|
| **P0** | 🔴 **关键**：游戏核心功能必需 | 第1-2周必须完成 |
| **P1** | 🟠 **重要**：功能完善性需要 | 第2-3周完成 |
| **P2** | 🟡 **可选**：增强体验的素材 | 第3周之后完成 |

## 🎨 制作工艺建议

### 工具推荐

| 工具 | 用途 | 说明 |
|------|------|------|
| **Photoshop / Procreate** | 美术绘制 | 原画创作、细节处理 |
| **Aseprite** | 动画制作 | 像素动画和帧动画制作 |
| **Spine / Dragonbones** | 骨骼动画 | 复杂动画（可选） |
| **TexturePacker** | 图集打包 | 优化加载性能（可选） |
| **TinyPNG / ImageOptim** | 图片压缩 | 无损压缩PNG/WebP |

### 风格指南

- **配色**：鲜艳、温暖、清洁主题（蓝、绿、橙、黄为主）
- **线条**：圆润边角，避免过度写实
- **透视**：2.5D立体感，轻微倾斜
- **细节**：Q版风格，夸张但不显脏
- **一致性**：所有素材风格统一，易于识别

### 质量标准

| 项目 | 标准 |
|------|------|
| **分辨率** | 逻辑像素标准 + 2倍DPR物理像素 |
| **透明度** | PNG保证通道，无锯齿 |
| **色彩空间** | sRGB，避免色差 |
| **文件大小** | 单个ICON < 20KB，场景背景 < 100KB |
| **边缘处理** | 抗锯齿处理，平滑过渡 |
| **动画帧数** | 3-5帧为宜，避免过多 |
| **命名规范** | 全小写、下划线分隔、语义清晰 |
